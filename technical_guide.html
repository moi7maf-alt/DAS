نظرة عامة على النظام
📌 ما هو النظام؟

نظام لوحة التخطيط التنموي هو تطبيق ويب تحليلي يعرض المؤشرات التنموية لـ 12 محافظة أردنية عبر 8 قطاعات، ويولّد تقارير استراتيجية تفاعلية قابلة للطباعة.

⚙️ طبيعة النظام التقنية
✅
نظام Static بالكامل — لا يحتاج قاعدة بيانات ولا خادم تطبيقات
النظام مبني من ملفات HTML + JSON + JavaScript فقط. يعمل على أي خادم ويب بسيط (IIS, Apache, Nginx) دون أي إعدادات معقدة.

إجراءات النشر على السيرفر
🪟 الطريقة أ: IIS على Windows Server
  1. فتح IIS Manager
    ابحث عن "Internet Information Services (IIS) Manager" في قائمة ابدأ وافتحه بصلاحيات المسؤول.
  2. إنشاء موقع أو مجلد افتراضي
    انقر بزر الماوس الأيمن على "Sites" ← "Add Website"، أو أضف "Virtual Directory" تحت موقع موجود.
  3. نسخ الملفات
    انسخ الملفات الثلاثة إلى مجلد الموقع على السيرفر:
# الملفات التي تُنسخ للسيرفر فقط src/index.html ← الواجهة الرئيسية src/data.json ← بيانات المحافظات src/strategic_plans.json ← الخطط الاستراتيجية
  1. تعيين الملف الافتراضي
    في IIS، انقر على "Default Document" وتأكد من وجود index.html في القائمة. إذا لم يكن موجوداً أضفه.
  2. التحقق من نوع MIME لـ JSON
    في IIS ← MIME Types، تأكد من وجود: Extension: .json | MIME Type: application/json. إذا لم يكن موجوداً أضفه.
  3. اختبار النشر
    افتح المتصفح وانتقل إلى عنوان الموقع. يجب أن تظهر لوحة التخطيط التنموي.
🐧 الطريقة ب: Apache/Nginx على Linux
# Apache — نسخ الملفات sudo mkdir -p /var/www/html/JLDA sudo cp src/index.html /var/www/html/JLDA/ sudo cp src/landing.html /var/www/html/JLDA/ sudo cp src/login.html /var/www/html/JLDA/ sudo cp src/chat.html /var/www/html/JLDA/ sudo cp src/upload.html /var/www/html/JLDA/ sudo cp src/audit.html /var/www/html/JLDA/ sudo cp src/data.json /var/www/html/JLDA/ sudo cp src/budget_data.json /var/www/html/JLDA/ sudo cp src/predictive_data.json /var/www/html/JLDA/ sudo cp src/strategic_comprehensive.json /var/www/html/JLDA/ # تعيين الصلاحيات sudo chown -R www-data:www-data /var/www/html/JLDA sudo chmod -R 755 /var/www/html/JLDA # إعادة تشغيل Apache sudo systemctl restart apache2
ℹ️
الوصول للموقع
بعد النشر يمكن الوصول عبر: http://[عنوان-السيرفر]/JLDA/
📁 الطريقة ج: مجلد مشترك على الشبكة الداخلية
  1. إنشاء مجلد مشترك
    أنشئ مجلداً على السيرفر وشاركه على الشبكة الداخلية بصلاحية القراءة لجميع الموظفين.
  2. نسخ جميع الملفات
    انسخ ملفات HTML و JSON من مجلد src/ إلى المجلد المشترك.
  3. توزيع الرابط
    أرسل للموظفين مسار الملف مثل: \\server\JLDA\index.html أو رابط HTTP إذا كان مجلداً ويب.
⚠️
ملاحظة مهمة للمجلد المشترك
بعض المتصفحات تمنع تحميل JSON من مسار ملف محلي. الحل: استخدم IIS أو Apache لخدمة الملفات عبر HTTP بدلاً من المسار المباشر.

استكشاف الأخطاء وإصلاحها
المشكلةالسبب المحتملالحل
الصفحة تفتح لكن لا تظهر بيانات ملف data.json غير موجود أو في مسار خاطئ تأكد أن data.json في نفس مجلد index.html على السيرفر

خطأ عند تشغيل generate_data.py اسم محافظة خاطئ في Excel أو عمود مفقود راجع أسماء المحافظات والأعمدة في الملف المذكور في رسالة الخطأ
ModuleNotFoundError: pandas المكتبات غير مثبتة شغّل: pip install pandas openpyxl numpy
محافظة لا تظهر بياناتها اسم المحافظة في Excel لا يطابق الأسماء المعتمدة راجع جدول الأسماء المعتمدة في القسم 5
خطأ 403 Forbidden على IIS صلاحيات المجلد غير صحيحة تأكد أن مستخدم IIS_IUSRS لديه صلاحية القراءة على المجلد
ملف JSON لا يُحمَّل (CORS error) فتح الملف مباشرة من المسار المحلي استخدم خادم ويب (IIS/Apache) بدلاً من فتح الملف مباشرة
الأرقام تظهر صفراً لمحافظة معينة خلايا فارغة أو تحتوي شرطة (-) في Excel استبدل الشرطات بـ 0 في ملف Excel المعني

أمان البيانات والحماية السيبرانية
🔒 إجراءات حماية الموقع أثناء النشر
⚠️
تنبيه أمني مهم
ملفات Data/input/ تحتوي على بيانات حساسة (رواتب، بطالة، مؤشرات ديموغرافية). لا تُرفع هذه الملفات أبدًا إلى السيرفر العام — تبقى على جهاز العمل فقط.
1. استخدام HTTPS
  1. شهادة SSL/TLS
    وفّر شهادة SSL من Let's Encrypt (مجانية) أو من جهة شهارات موثوقة. على IIS: استخدم ميزة "Server Certificates" و"Bindings" لربط الشهادة بالموقع.
  2. إعادة التوجيه الإجباري
    اضبط قاعدة إعادة توجيه HTTP → HTTPS في ملف web.config (IIS) أو إعدادات الخادم (Apache/Nginx).
2. التحكم بالوصول
  1. المصادقة الأساسية (Basic Auth)
    على IIS: فعّل "Basic Authentication" أو استخدم web.config لتحديد مستخدمين مسموحين:
    <authorization><allow users="admin"/><deny users="*"/></authorization>
  2. تقييد الوصول بالـ IP
    اضبط جدار الحماية ليُسمح بالوصول فقط من عناوين IP موثوقة (شبكة الوزارة / VPN).
  3. صلاحيات الملفات
    على السيرفر: منح صلاحية القراءة فقط لحساب الخدمة (IIS_IUSRS / www-data). منع الكتابة إلا لمجلدات السجلات.
3. حماية الملفات الحساسة
الملفمستوى الحساسيةإجراء الحماية
data.jsonمتوسط — يحتوي مؤشرات أداءلا يحتاج تشفير خاص (بيانات عامة)
budget_data.jsonمرتفع — بيانات ماليةتشفير AES-256 أو حماية بصلاحيات NTFS
strategic_plans.jsonمرتفع — خطط استراتيجيةتشفير أو تقييد الوصول عبر IP
predictive_data.jsonمتوسط — تنبؤاتلا يحتاج تشفير خاص
Data/input/*.xlsxحرج — بيانات مصدر خاملا يُرفع أبدًا للسيرفر — يبقى على جهاز العمل
4. النسخ الاحتياطي الآمن
  • تخزين النسخ الاحتياطية في موقع جغرافي منفصل
  • تشفير النسخ الاحتياطية باستخدام 7-Zip أو GPG (AES-256)
  • جدولة نسخ احتياطية أسبوعية على الأقل
  • اختبار استرداد النسخ الاحتياطية شهرياً
  • الاحتفاظ بـ 3 نسخ احتياطية على الأقل (قاعدة 3-2-1)
5. إعدادات جدار الحماية (Firewall)
# تفعيل HTTPS فقط (منع HTTP) # IIS: إضافة قاعدة في Windows Firewall netsh advfirewall firewall add rule name="HTTPS Only" dir=in action=allow protocol=TCP localport=443 netsh advfirewall firewall add rule name="Block HTTP" dir=in action=block protocol=TCP localport=80 # Linux: iptables sudo iptables -A INPUT -p tcp --dport 443 -j ACCEPT sudo iptables -A INPUT -p tcp --dport 80 -j DROP
6. تسجيل الأنشطة (Audit Logging)
ℹ️
تفعيل سجلات الوصول
فعّل IIS Logging أو Apache Access Log لتسجيل كل طلبات الوصول للموقع. احتفظ بالسجلات لمدة 90 يوماً على الأقل.

دليل النشر السريع (ملخص)
✅
اختصار خطوات النشر الكامل
هذا الملخص يغطي جميع الخطوات اللازمة لنشر الموقع لأول مرة أو تحديثه.
  1. التهيئة الأولية (مرة واحدة فقط)
    على جهاز العمل: تثبيت Python 3.9+ والمكتبات المطلوبة:
    pip install pandas openpyxl numpy

    على السيرفر: تثبيت IIS/Apache/Nginx، تفعيل دعم JSON MIME type، وتثبيت شهادة SSL.
  2. بناء النظام
    شغّل الأمر الموحد من مجلد المشروع:
    python src/run_all.py

    أو الأوامر الفردية:
    python src/generate_data.py → معالجة البيانات
    python src/build_budget.py → حساب الموازنة
    python src/build_predictive2.py → التحليل التنبؤي
    python src/write_dashboard.py → بناء الهيكل
    python src/inject_budget.py → حقن تبويب الموازنة
    python src/inject_solutions.py → حقن تبويب الحلول
    python src/inject_predictive.py → حقن تبويب التنبؤي
    python src/inject_ai.py → حقن تبويب التقرير
    python src/build_landing.py → صفحة الهبوط
    python src/build_guide.py → بناء الدليل
  3. التحقق المحلي
    افتح src/index.html في متصفح Chrome/Edge وتحقق من: ظهور جميع المحافظات الـ12، عمل الرسوم البيانية، ظهور تبويب "مشاريع مقترحة" بالبطاقات.
  4. رفع الملفات للسيرفر
    انسخ الملفات التالية إلى مجلد السيرفر:
    src/index.html (الملف الرئيسي ~2.4 MB)
    src/data.json (بيانات المحافظات)
    src/strategic_plans.json (الخطط والمشاريع)
    src/budget_data.json (بيانات الموازنة)
    src/predictive_data.json (البيانات التنبؤية)
    src/landing.html (صفحة الهبوط)

    ملاحظة: لا تنقل مجلد Data/input/ إلى السيرفر — يبقى على جهاز العمل فقط.
  5. إعدادات السيرفر
    تعيين index.html كملف افتراضي، التحقق من دعم MIME لنوع .json، تفعيل HTTPS، تقييد الوصول بالـ IP عند الحاجة.
  6. الاختبار النهائي
    افتح الموقع من متصفح مختلف عن بُعد (أو من جهاز آخر)، تحقق من الطباعة (Ctrl+P)، وتأكد من عرض جميع التبويبات.
⚠️
تنبيهات مهمة
• ملف index.html الناتج كبير (~2.4 MB) بسبب البيانات المدمجة — لا يؤثر على الأداء لأنه ملف ثابت.
• عند التحديث: كرّر الخطوة 1 فقط (run_all.py) ثم انسخ الـ 5 ملفات للسيرفر مرة أخرى.
• خادم Node.js (serve.js) مخصص للتطوير فقط — للاستخدام الإنتاجي استخدم IIS/Apache/Nginx.
