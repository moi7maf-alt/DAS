# النشر الآمن والاستضافة المستقلة

هذا المستند يوضح كيفية تحضير المشروع للنشر على سيرفر مستقل مع الحفاظ على سيادة البيانات وأمان التشغيل.

## 1. هدف الحزمة
- إنشاء حزمة جاهزة للنشر تحتوي على الموقع الثابت `dist/`.
- تضمين كل ملفات مصدر البناء (`src/`) وبيانات المشروع.
- الحفاظ على عزل البيانات وعدم نشر بيئات التطوير.

## 2. محتويات الحزمة الجاهزة
- `dist/` — موقع ثابت جاهز للرفع إلى خادم ويب.
- `src/` — جميع السكريبتات وملفات البيانات التي تُستخدم لتحديث وبناء الموقع.
- `requirements.txt` — مكتبات Python المطلوبة لتحديث البيانات وإعادة الإنشاء.
- `deploy.py` — سكريبت تجميع الموقع.
- `prepare_server_package.py` — سكريبت إنشاء حزمة النشر الآمنة.
- `HOW_TO_ADD_DATA.md` — توجيهات إضافة أو تحديث البيانات.

## 3. نموذج النشر المقترح
### الخيار 1: استضافة ثابتة فقط
1. شغّل `python deploy.py` لإنشاء `dist/`.
2. انسخ محتويات `dist/` إلى مجلد الجذر الخاص بالموقع على السيرفر.
3. تأكد أن الملف الافتراضي هو `index.html`.
4. فعل HTTPS، واغلق الوصول إلى مجلدات التطوير.

### الخيار 2: نشر مع إمكانية التحديث لاحقاً
1. انسخ `server_package/` أو المشروع الأصلي إلى السيرفر الآمن.
2. ثبّت Python 3.9+ وفعّل بيئة افتراضية في مكان آمن.
3. شغل:
   ```bash
   python -m pip install -r requirements.txt
   ```
4. لتحديث المحتوى، شغّل:
   ```bash
   python src/run_all.py
   python deploy.py
   ```
5. انسخ `dist/` النهائي إلى جذر الموقع.

## 4. شروط الأمان
- استعمل HTTPS/TLS لجميع الطلبات.
- لا ترفع `venv/` أو `venv_packages/` أو أي ملفات تطوير إلى الخادم.
- ضع صلاحيات قراءة فقط لمجلد `dist/` من قبل خدمة الويب.
- عزل ملفات البيانات الحساسة (`src/*.json`) عن الوصول العام إذا لم يتم استخدامها مباشرة في واجهة المستخدم.
- عطل استعراض المجلدات (Directory Listing) في خادم الويب.
- ضع قيوداً على MIME types: تأكد من أن `.json` يعالج كـ `application/json` و `.html` كـ `text/html`.

## 5. حماية سيادة البيانات
- جميع ملفات البيانات تحت تحكمك في `src/`.
- لا تعتمد على خدمات خارجية لمعالجة البيانات الحساسة.
- إذا احتجت مكتبة Chart.js دون اتصال إنترنت، استخدم `src/chart.min.js` محلياً بدل CDN.
- احتفظ بنسخ احتياطية مشفرة للبيانات الأصلية.

## 6. نصائح إضافية
- استخدم جدار حماية (Firewall) لحظر المنافذ غير الضرورية.
- استخدم حساب مستخدم منفصل لخدمة الويب.
- إذا كان السيرفر يستخدم Windows، قم بتنفيذ التحديثات الأمنية دورياً.
- لا تسمح بتنفيذ أو تحميل ملفات `.py` من مجلد `dist/`.
